git clone
 يُستخدم لإنشاء نسخة محلية من مستودع (Repository) موجود. هذا المستودع يمكن أن يكون موجودًا على الإنترنت (مثل GitHub، GitLab، أو Bitbucket) أو على جهاز آخر في الشبكة المحلية.

الغرض من git clone 
نشاء نسخة كاملة من مستودع يحتوي على جميع الملفات الموجودة فيه.
نسخ تاريخ التعديلات بالكامل (History) بما في ذلك الإصدارات السابقة.
الربط بالمستودع الأصلي لتحديثات مستقبلية (المستودع الأصلي يُسمى remote origin).

الصيغة العامة:
git clone <repository-url> [destination-folder]

أمثلة:
نسخ مستودع باستخدام HTTPS:

git clone https://github.com/user/repo.git



git push
يُستخدم لإرسال التغييرات التي أجريتها على المستودع المحلي إلى المستودع البعيد (remote repository).



الغرض من git push:
مزامنة التغييرات من المستودع المحلي إلى المستودع البعيد
مشاركة التحديثات مع أعضاء الفريق الذين يعملون على نفس المشروع
تحديث الفروع (branches) البعيدة بالتعديلات المحلية


الصيغه العامه 
git push <remote-name> <branch-name>

أمثلة:
إرسال التغييرات إلى الفرع الافتراضي (عادةً main أو master):

git push origin main
